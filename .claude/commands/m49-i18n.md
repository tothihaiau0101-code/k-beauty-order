# M49 - i18n Implementation Guide

## Overview
Implement internationalization (i18n) support for the k-beauty-order application with English (EN) and Vietnamese (VI) language toggle.

## Requirements

### 1. Create i18n Configuration
- Create `src/i18n/translations.js` with all translatable strings
- Create `src/i18n/i18n.js` with i18n helper functions
- Support EN and VI languages

### 2. Language Toggle Component
- Add language toggle button to Navbar
- Display current language (EN/VI)
- Switch between languages on click

### 3. Language Persistence
- Store selected language in localStorage
- Load saved language on app initialization
- Default to 'vi' if no saved preference

### 4. Content Swapping
- Use data-i18n attributes on HTML elements
- Create function to swap text content based on current language
- Update all UI text dynamically

## Implementation Steps

1. Create translation files with all UI strings
2. Create i18n helper module
3. Add language toggle to Navbar
4. Implement language switching logic
5. Add data-i18n attributes to all translatable elements
6. Test language persistence and switching

## File Structure
```
src/
├── i18n/
│   ├── translations.js
│   └── i18n.js
├── components/
│   └── Navbar.jsx (updated)
└── App.jsx (updated with i18n init)
```
